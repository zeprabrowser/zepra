NXHTTP TLS/SSL Implementation

Version: 0.0.1 | Last Updated: December 2025 | Status: Production Ready

Overview

NXHTTP is ZepraBrowser's native HTTP client library with full TLS/SSL support. It provides secure HTTPS connections using OpenSSL, integrated via the SSLContext wrapper in the networking module.

Note: ZepraBrowser uses KetiveeSearch (powered by ZepraSearch) as the default search engine. All search queries are routed to https://ketivee.com/search with full TLS encryption.

TLS Features

Feature Status Description
TLS 1.2 Enabled Minimum supported version (industry standard)
TLS 1.3 Enabled Maximum version with 0-RTT support
SNI (Server Name Indication) Enabled Required for virtual hosting and CDNs
Certificate Verification Enabled System CA bundle loaded automatically
Hostname Verification Enabled Prevents MITM attacks
Session Resumption Enabled Faster reconnections to same host
ALPN (HTTP/2) Negotiated Advertises h2 and http/1.1
OCSP Stapling Enabled Certificate revocation checking

Cipher Suites

NXHTTP uses modern AEAD cipher suites only. No CBC mode is allowed.

TLS 1.3 Ciphers

TLS_AES_256_GCM_SHA384
TLS_CHACHA20_POLY1305_SHA256
TLS_AES_128_GCM_SHA256

TLS 1.2 Ciphers

ECDHE-ECDSA-AES256-GCM-SHA384
ECDHE-RSA-AES256-GCM-SHA384
ECDHE-ECDSA-CHACHA20-POLY1305
ECDHE-RSA-CHACHA20-POLY1305
ECDHE-ECDSA-AES128-GCM-SHA256
ECDHE-RSA-AES128-GCM-SHA256

Usage

Basic HTTPS Request

// In zepra_browser.cpp
HttpResponse resp = httpGet("https://ketivee.com/search?q=test");

if (resp.success) {
    std::cout << "Content: " << resp.data << std::endl;
    std::cout << "Status: " << resp.statusCode << std::endl;
} else {
    std::cerr << "Error: " << resp.error << std::endl;
}

Configuring SSL Verification

NxHttpClientConfig config = {0};
config.connect_timeout_ms = 10000;
config.read_timeout_ms = 10000;
config.follow_redirects = true;
config.max_redirects = 10;
config.verify_ssl = true;  // Enable certificate verification
config.user_agent = "ZepraBrowser/1.0 (NeolyxOS)";

NxHttpClient* client = nx_http_client_create(&config);

Certificate Verification

The SSL context automatically loads system CA certificates from these locations:

/etc/ssl/certs/ca-certificates.crt    (Debian/Ubuntu)
/etc/pki/tls/certs/ca-bundle.crt      (RHEL/CentOS)
/etc/ssl/ca-bundle.pem                 (OpenSUSE)
/etc/pki/ca-trust/extracted/pem/tls-ca-bundle.pem

Verification Results

Result Meaning
CertVerifyResult::OK Certificate is valid
CertVerifyResult::EXPIRED Certificate has expired
CertVerifyResult::HOSTNAME_MISMATCH Certificate doesn't match requested hostname
CertVerifyResult::UNTRUSTED_ROOT Root CA not in trusted store
CertVerifyResult::SELF_SIGNED Self-signed certificate (not trusted)

Error Handling

NxHttpError err;
NxHttpResponse* res = nx_http_client_send(client, req, &err);

if (!res) {
    switch (err) {
        case NX_HTTP_ERROR_SSL:
            // TLS handshake failed or certificate error
            std::cerr << "TLS Error: " 
                      << Zepra::Networking::SSLContext::getLastError() 
                      << std::endl;
            break;
        case NX_HTTP_ERROR_DNS:
            std::cerr << "DNS resolution failed" << std::endl;
            break;
        case NX_HTTP_ERROR_CONNECT:
            std::cerr << "Connection refused" << std::endl;
            break;
    }
}

Build Requirements

# CMake configuration (automatic)
find_package(OpenSSL REQUIRED)
target_link_libraries(nxhttp PRIVATE networking OpenSSL::SSL OpenSSL::Crypto)

# Install OpenSSL (if not present)
# Ubuntu/Debian:
sudo apt install libssl-dev

# RHEL/CentOS:
sudo dnf install openssl-devel

Security Best Practices

Production Configuration:
  • Always keep verify_ssl = true in production
  • Never accept self-signed certificates without user consent
  • Keep OpenSSL updated for security patches
  • Monitor for certificate expiration on your domains
